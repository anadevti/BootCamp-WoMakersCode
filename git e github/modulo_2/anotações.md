# GitHub: Comandos e Funcionalidades

## 1. Git e Comandos Básicos
- **`git clone [URL]`**: Clona um repositório para sua máquina local.
- **`git pull`**: Atualiza seu repositório local com as últimas mudanças do repositório remoto.
- **`git push`**: Envia suas mudanças locais para o repositório remoto.
- **`git commit -m "mensagem"`**: Salva suas mudanças no repositório local com uma mensagem descrevendo o que foi feito.
- **`git add [arquivo]`**: Adiciona um arquivo específico ao próximo commit.
- **`git status`**: Mostra o status dos arquivos no diretório de trabalho e no índice.

## 2. Branching e Merging
- **`git branch [nome-da-branch]`**: Cria uma nova branch.
- **`git checkout [nome-da-branch]`**: Muda para a branch especificada.
- **`git merge [nome-da-branch]`**: Mescla as mudanças de uma branch na branch atual.

## 3. Usabilidades do GitHub
- **Pull Requests**: Uma maneira de contribuir para um projeto, solicitando que suas mudanças sejam revisadas e mescladas no repositório principal.
- **Issues**: Usadas para rastrear tarefas, melhorias e bugs para um projeto.
- **GitHub Actions**: Automatiza fluxos de trabalho, incluindo CI/CD, sempre que um evento no repositório ocorre (como push, pull request, etc.).
- **GitHub Pages**: Hospeda sites estáticos diretamente de um repositório do GitHub.
- **Wiki**: Fornece um local para manter a documentação do projeto.

## 4. Comandos Avançados
- **`git rebase [branch-base]`**: Aplica mudanças de uma branch sobre outra.
- **`git stash`**: Salva mudanças locais temporariamente sem commitar.
- **`git fetch`**: Baixa objetos e refs de outro repositório.
